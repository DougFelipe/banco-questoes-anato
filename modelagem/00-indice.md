# 00 - Índice da Modelagem

> **LeetCode YAML Study App - Documentação de Padrões**

Este diretório contém a modelagem e padronização para criação de novos problemas no sistema.

## 📚 Estrutura dos Documentos

| Arquivo | Descrição |
|---------|-----------|
| [01-visao-geral.md](./01-visao-geral.md) | Visão geral do projeto e estrutura de diretórios |
| [02-schema-problem.md](./02-schema-problem.md) | Schema completo do `problem.yaml` |
| [03-schema-solution.md](./03-schema-solution.md) | Schema completo dos arquivos de solução |
| [04-diretrizes-codigo.md](./04-diretrizes-codigo.md) | Diretrizes para implementação de código |
| [05-checklist.md](./05-checklist.md) | Checklist de implementação de novo problema |
| [06-templates.md](./06-templates.md) | Templates prontos para copiar |

## 🎯 Objetivo

Padronizar a criação de novos problemas para manter:
- **Consistência** entre todos os problemas
- **Qualidade** na documentação e código
- **Escalabilidade** para adicionar mais linguagens/paradigmas
- **Manutenibilidade** facilitada

## 🚀 Quick Start

1. Leia o [01-visao-geral.md](./01-visao-geral.md) para entender a estrutura
2. Use o [05-checklist.md](./05-checklist.md) como guia passo-a-passo
3. Copie templates do [06-templates.md](./06-templates.md)
4. Valide schemas com [02-schema-problem.md](./02-schema-problem.md) e [03-schema-solution.md](./03-schema-solution.md)
